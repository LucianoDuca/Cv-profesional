02Proyectos
01

Instituto San Vicente

Desarrollo completo de una plataforma web institucional multipage para un colegio privado argentino, que incluye landing page, secciones académicas por nivel (Inicial, Primaria y Secundaria), área de inglés, galería de fotos, formulario de contacto y un portal deportivo (Club Sanvi) con múltiples disciplinas. El proyecto integra autenticación de usuarios, paneles de gestión diferenciados por rol (administradores, directivos y docentes), soporte multilenguaje (español/inglés) y optimización de assets para producción. Desplegado en Netlify con servidor propio en Node.js para endpoints sensibles.

HTML5 — estructura semántica y accesibilidad (ARIA)CSS3 — diseño responsive mobile-first, animaciones con scroll, variables CSS y media queriesJavaScript (ES6+) — lógica del cliente, Intersection Observer API, galería lightbox con navegación por tecladoNode.js + Express.js — servidor backend para manejo de rutas y endpoints de correoSupabase — base de datos PostgreSQL, autenticación y almacenamiento en la nubeResend API — envío transaccional de emailsi18n personalizado — sistema de internacionalización bilingüe sin librerías externasSEO on-page — metatags, Open Graph y optimización de crawlabilitySharp — procesamiento y conversión de imágenes a formato WebP para rendimientoNetlify — CI/CD, cabeceras HTTP personalizadas y despliegue continuoGit — control de versiones y gestión del flujo de trabajo

02

GoRugby

Aplicación web full-stack orientada a la comunidad rugbística, con seguimiento de partidos en vivo, predicciones, rankings de torneos globales y gestión de clubes. Integra múltiples APIs deportivas externas con un proxy seguro del lado del servidor para proteger credenciales. Soporta temas claro/oscuro, diseño responsive y sistema de roles (usuario/admin).

React 18 + TypeScript — SPA con tipado estricto, componentización modular y Context API para manejo de estado global (autenticación, tema, datos en vivo)Vite — Build tool con hot reload, configuración de proxies de desarrollo y optimización de bundles para producciónCSS personalizado (Design System) — Sistema de diseño propio con variables CSS, soporte de temas dinámicos (claro/oscuro/sistema), 6 paletas de acento y tamaños de fuente configurables sin recarga de páginaNode.js + Vercel Edge Functions — Funciones serverless en el edge para proxy de APIs externas, manejo de headers de autenticación y caching HTTP (stale-while-revalidate)Integración de APIs REST — Consumo de 4 APIs deportivas simultáneas (api-sports.io, ESPN, Highlightly, TheSportsDB) con normalización de datos, deduplicación y mapeo de IDs entre fuentesLocalStorage & gestión de sesión — Persistencia de preferencias, historial de predicciones, sesión de usuario y estado de la app sin base de datosGit & GitHub — Control de versiones, gestión de ramas y variables de entorno separadas por entorno (.env.local / Vercel env vars)Vercel — Deploy continuo con configuración de SPA routing, assets estáticos y funciones edge en producciónDiseño responsive — Navegación adaptativa (sidebar fijo en desktop, navbar inferior en mobile), layout fluido con CSS Grid y Flexbox

03

Fiambrería Duca

Aplicación de escritorio desarrollada para una fiambrería local, que permite consultar precios en tiempo real mediante escaneo de código de barras. El negocio puede gestionar su catálogo completo de productos desde la misma herramienta, con sincronización automática a la nube.

React + TypeScript — Interfaz construida con componentes funcionales, hooks (useState, useEffect, useMemo, useRef) y tipado estático para reducir errores en tiempo de desarrolloElectron — Empaqueta la app web como aplicación nativa de Windows, con instalador .exe generado automáticamente mediante electron-builderSupabase (PostgreSQL) — Base de datos en la nube como backend principal; maneja el CRUD completo de productos con consultas paginadas y filtros por categoría, nombre y código de barrasVite — Bundler moderno para compilación del frontend, con integración directa al proceso de Electron en modo desarrolloNode.js — Runtime del proceso principal de Electron; gestiona la comunicación entre la ventana del navegador embebido y el sistema operativoWeb Audio API — Feedback sonoro nativo del navegador al escanear un producto (tono de éxito o error), sin dependencias externas de audioCSS personalizado — Diseño responsivo con soporte de modo oscuro/claro, gestionado mediante clases condicionales en React
